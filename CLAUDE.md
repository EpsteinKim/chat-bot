# chat-bot — AI 챗봇 서비스

## 기술 스택

- 필수: Kotlin, Spring Boot
- DB: PostgreSQL
- 마이그레이션: Flyway (Hibernate 는 validate 만)
- 인증: JWT (stateless, 서버 측 블랙리스트 없음)
- LLM: OpenAI

## 코딩 규칙

- 코드에 주석을 작성하지 않는다.

## 실행 환경

- JDK 21 (Gradle toolchain 지정)
- PostgreSQL: `docker compose up -d` 후 `./gradlew bootRun`
- 접속 정보 override: `DB_URL`, `DB_USERNAME`, `DB_PASSWORD`

# 요구사항

## 1. 사용자 관리 및 인증

### user 데이터 (필수 요소)

- 이메일 (string)
- 패스워드 (string)
- 이름 (string)
- 생성일시 (timestamptz)
- 권한 (string): member 또는 admin

### 기능

- 회원 가입
  - 필수 입력: 이메일, 패스워드, 이름
- 로그인
  - 필수 입력: 이메일, 패스워드
  - 로그인 완료 시 JWT 형식의 토큰 발급
- 인증
  - 회원가입 및 로그인을 제외한 모든 요청은 JWT 토큰을 확인하여 인증한다.

## 2. 대화(chat) 관리

### chat 데이터 (필수 요소)

- 질문 (string)
- 답변 (string)
- 생성일시 (timestamptz)
- 스레드 (thread)

### 스레드(thread) 정의

- 응답 생성을 위해 OpenAI에 요청을 전송할 때 지난 대화 목록을 함께 보낸다. 이 목록 단위를 스레드라 정의한다.
- 스레드는 유저마다 여러 개 생성 가능하다. (thread : user = N : 1)
- 스레드 생성 시점
  - 해당 유저의 첫 질문이거나, 마지막 질문 후 30분이 지난 시점에 질문할 경우 새 스레드를 생성한다.
  - 마지막 질문으로부터 30분 이내에 다시 질문할 경우 기존 스레드를 유지한다.

### 기능

- 대화 생성
  - 질문을 입력받고 생성된 답변을 응답한다.
  - 추가 옵션 파라미터
    - `isStreaming` (boolean): true일 경우 stream 형태로 응답한다.
    - `model` (string): 설정될 경우 해당 모델로 응답을 생성한다.
- 대화 목록 조회
  - 요청한 유저의 모든 대화를 응답한다.
  - 스레드 단위로 그룹화된 대화 목록을 응답한다.
  - 각 유저는 자신이 생성한 스레드와 대화만 조회 가능, 관리자는 모든 스레드와 대화 조회 가능.
  - 생성일시 기준 오름차순/내림차순 정렬 가능, 페이지네이션 가능.
- 스레드 삭제
  - 특정 스레드를 선택하여 삭제한다.
  - 각 유저는 자신이 생성한 스레드만 삭제 가능하다.

## 3. 사용자 피드백 관리

### feedback 데이터 (필수 요소)

- 사용자 ID (string)
- 대화 ID (string): 피드백 대상 대화의 ID
- 긍정/부정 유무 (boolean)
- 생성일시 (timestamptz)
- 상태 (string): pending, resolved 중 하나

### 기능

- 피드백 생성
  - 특정 대화에 대한 피드백을 생성한다.
  - 각 사용자는 자신이 생성한 대화에만 생성 가능, 관리자는 모든 대화에 생성 가능.
  - 각 사용자는 하나의 대화에 오직 하나의 피드백만 생성 가능하다. 즉 하나의 대화에는 서로 다른 사용자들이 생성한 n개의 피드백이 존재할 수 있다.
- 피드백 목록 조회
  - 각 사용자는 자신이 생성한 피드백만 조회 가능, 관리자는 모든 피드백 조회 가능.
  - 생성일시 기준 오름차순/내림차순 정렬 가능, 페이지네이션 가능.
  - 긍정/부정 유무로 필터링 가능.
- 피드백 상태 변경
  - 관리자는 피드백의 상태를 업데이트할 수 있다.

## 4. 분석 및 보고

### 기능

- 사용자 활동 기록 요청
  - 회원가입, 로그인, 대화 생성 수를 응답한다.
  - 요청 시점으로부터 하루 동안의 기록을 응답한다.
  - 관리자만 요청 가능하다.
- 보고서 생성 요청
  - 호출 시 CSV 형태의 보고서를 생성한다.
  - 보고서에는 모든 사용자의 대화 목록과 어떤 사용자가 생성했는지 포함된다.
  - 요청 시점으로부터 하루 동안의 보고서가 생성되어야 한다.
  - 관리자만 요청 가능하다.
